--------------------
동기(Synchronous)
--------------------
코드가 위에서 아래로 순차적으로 실행 == 약속한 것
이전 작업이 끝나야 다음 작업이 실행 == 예외> 호이스팅
실행 흐름이 직관적이지만 처리시간이 오래 걸리는 작업이 있으면 전체 프로그램이 지연 될 수 있음

--------------------
비동기(ASynchronous)
--------------------
특정 작업이 끝날 때까지 기다리지 않고 다음 코드가 먼저 실행됨
시간이 오래 걸리는 작업(예: 네트워크 요청, 파일 읽기 등)을 수행할 때 유용함
콜백(callback), 프로미스(Promise), async/await 등을 이용해 처리함


++
Sync(동기) : 결과물을 만들어 내기 위해 사전에 정의한 ★약속★ 
절차지향 (순서대로 진행되는 것 == 약속)
Clinet(브라우저)가 하나의 페이지를 요청을 하면 WebServer에서 이에 대한 응답을 하기 전까지는 웹 페이지에서 다른 것을 하지 못함

++
비동기 : (약속을 깨는 것, 절차를 깬다.) + 스레드: 컴퓨터 내에 처리를 해주는 단위

= WebServer에서 여러개의 요청이 가능하고 응답이 오는 것과 상관없이 다른 작업이 가능함
= 비동기 방식으로 서버를 구현해야 웹브라우저에 한정되지 않고 모바일 디바이스 등과 같은 기타 디바이스에서 사용 가능
= 요청에 대한 응답을 주든 안 주든 다른 작업이 가능함
= 따로 작업하는 시기가 분리된다.
= 코드적 의미의 비동기 : 하나의 스레드를 더 만듦(==작업흐름을 새로 만듦으로써 바로 작업을 선위할 수 있도록 도와줌)
= 독자적 내용 처리를 위해 필요함
